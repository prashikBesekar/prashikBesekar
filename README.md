`Nov 2025 – Jan 2026`

```

  GitHub Repo ──push──▶ EC2 Instance ──▶ Nginx Reverse Proxy ──▶ Live Website
                            │
                     Security Groups
                     (port 80/443 only)
```

**What I built & solved:**

- 🔧 Configured **Nginx** with custom server blocks — serving a live, publicly accessible site
- 🔄 Built a **Git-based CI/CD pipeline** — push to GitHub, changes auto-reflect on production server
- 🛡️ Hardened **AWS Security Groups** — only necessary ports exposed, everything else locked
- 🐛 Debugged critical production issues: port conflicts, file permission errors, service crashes
- 📋 Wrote deployment runbooks — reproducible setup from scratch on a fresh EC2 instance

> This wasn't a tutorial. This was a real production environment I built, broke, and fixed myself.
